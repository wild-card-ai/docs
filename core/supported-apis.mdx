## Detailed API Coverage

### Gmail
- **Messages**: Send, receive, list, and manage emails
- **Drafts**: Create, update, and manage draft emails
- **Labels**: Organize emails with custom labels
- **Threads**: Manage email conversations
- **History**: Track mailbox changes
- **Settings**: Configure email settings

### Airtable
- **Records**: CRUD operations on table records
- **Fields**: Manage table structure
- **Views**: Create and manage different views
- **Attachments**: Handle file attachments
- **Comments**: Collaborate on records
- **Webhooks**: Set up real-time notifications

### Slack
- **Messages**: Send and manage channel messages
- **Channels**: Create and manage channels
- **Users**: User management and information
- **Files**: Share and manage files
- **Reactions**: Handle message reactions
- **Search**: Search messages and files

## Example Usage

Here's how to use different APIs with Wildcard:

```python
from wildcard_core.client import WildcardClient
from wildcard_core.models.Action import Action
from wildcard_core.models import APIService

async def main():
    client = WildcardClient(api_key="your_api_key")
    
    # Gmail Example
    await client.run_tool_with_args(
        tool_name=Action.Gmail.MESSAGES_SEND,
        to="recipient@example.com",
        subject="Hello",
        body="Test email"
    )
    
    # Airtable Example
    await client.run_tool_with_args(
        tool_name=Action.Airtable.RECORDS_LIST,
        base_id="your_base_id",
        table_name="your_table"
    )
    
    # Slack Example
    await client.run_tool_with_args(
        tool_name=Action.Slack.CHAT_POST_MESSAGE,
        channel="general",
        text="Hello from Wildcard!"
    )

if __name__ == "__main__":
    asyncio.run(main())
```
